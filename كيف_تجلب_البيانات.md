# كيفية جلب البيانات من Fustog API

## 🚀 الطرق المتاحة

### الطريقة 1: السكربت البسيط (الأسهل) ⭐

```bash
python scripts\fetch_data_simple.py
```

**ماذا يفعل:**
- يجلب التصنيفات
- يجلب جميع المنتجات
- يجلب الأسعار
- يحفظ كل شيء في ملف JSON واحد

**النتيجة:**
- ملف في `data/fustog_data_YYYY_MM_DD_HHMMSS.json`

---

### الطريقة 2: السكربت الشامل (الأقوى)

```bash
python scripts\fetch_all_data.py
```

**ماذا يفعل:**
- يجلب كل البيانات المتاحة
- يجلب التصنيفات والمنتجات والأسعار
- يجلب المدن والمتاجر (إن وجدت)
- يجلب إعدادات التاجر

**خيارات إضافية:**

```bash
# حفظ كل نوع في ملف منفصل
python scripts\fetch_all_data.py --individual

# عدم جلب المتاجر
python scripts\fetch_all_data.py --no-stores

# عدم جلب المدن
python scripts\fetch_all_data.py --no-cities

# تحديد API URL مخصص
python scripts\fetch_all_data.py --base-url "https://api.fustog.app/api/v1"

# تحديد API Key
python scripts\fetch_all_data.py --api-key "your_key_here"
```

**النتيجة:**
- ملف شامل أو ملفات منفصلة في `data/`

---

### الطريقة 3: السكربت الأصلي (مع Excel)

```bash
python scripts\scrape_products_prices_compare.py
```

**ماذا يفعل:**
- يجلب البيانات
- يقارن الأسعار
- يصدر CSV و Excel
- يحسب إحصائيات

**النتيجة:**
- ملفات CSV و Excel في `data/`

---

## 📋 ما الذي يتم جلبه؟

### البيانات الأساسية:
1. **التصنيفات** (`/category/Categories`)
2. **المنتجات** (`/product/ProductsByCategory`)
3. **الأسعار** (`/product/ItemsPrices`)

### بيانات إضافية (إن وجدت):
4. **المدن** (`/city/Cities` أو `/cities`)
5. **المتاجر** (`/store/Stores`)
6. **الإعدادات** (`/retailer/Settings`)

## 🔧 استكشاف الأخطاء

### خطأ في الاتصال:
```
[ERROR] خطأ في جلب التصنيفات: ...
```

**الحل:**
- تحقق من الاتصال بالإنترنت
- تحقق من صحة BASE_URL
- تحقق من وجود API Key (إن كان مطلوباً)

### لا توجد بيانات:
```
[WARN] لم يتم العثور على endpoint للمدن
```

**الحل:**
- هذا طبيعي - بعض الـ endpoints قد لا تكون متاحة
- البيانات الأساسية (تصنيفات، منتجات، أسعار) يجب أن تعمل
