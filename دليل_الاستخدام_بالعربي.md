# دليل استخدام أدوات الهندسة العكسية - بالعربي

## 🎯 ماذا يمكنك أن تفعل؟

### 1. **اكتشاف جميع API Endpoints**
   - معرفة جميع الروابط المتاحة في API
   - معرفة نوع الطلب (GET, POST, PUT, DELETE)
   - معرفة المعاملات المطلوبة لكل رابط

### 2. **فهم بنية البيانات**
   - معرفة شكل البيانات المرسلة والمستقبلة
   - معرفة أنواع الحقول (نص، رقم، قائمة، إلخ)
   - معرفة الحقول المطلوبة والاختيارية

### 3. **تحليل آلية المصادقة (Authentication)**
   - معرفة كيف يتم تسجيل الدخول
   - معرفة نوع التوكن المستخدم
   - معرفة الـ Headers المطلوبة

### 4. **إنشاء توثيق تلقائي**
   - توثيق كامل للـ API
   - ملف OpenAPI يمكن استخدامه في Postman
   - تقارير تحليلية

## 🚀 الخطوات العملية

### الخطوة 1: التقاط حركة المرور (Capture Traffic)

**الطريقة الأولى - استخدام mitmproxy:**
```bash
# افتح PowerShell في مجلد المشروع
cd fustoog\sandbox\ingest
.\run_mitmproxy.ps1
```

**بعد تشغيل mitmproxy:**
1. افتح المتصفح
2. اضبط الـ Proxy على `localhost:8080`
3. استخدم التطبيق أو الموقع بشكل طبيعي
4. كل الطلبات ستُسجل تلقائياً

**الطريقة الثانية - استخدام Browser DevTools:**
1. افتح Developer Tools (F12)
2. اذهب لتبويب Network
3. استخدم التطبيق
4. انقر بزر الماوس الأيمن على Network tab
5. اختر "Save all as HAR"

### الخطوة 2: تحليل البيانات

**تشغيل التحليل السريع:**
```bash
python scripts\quick_reverse_engineer.py
```

هذا الأمر سيقوم بـ:
- ✅ تحليل ملف HAR الموجود
- ✅ البحث في الكود عن API calls
- ✅ إنشاء التوثيق الكامل
- ✅ إنشاء ملف OpenAPI

### الخطوة 3: مراجعة النتائج

افتح الملفات المولدة في مجلد `data/`:

1. **`api_documentation.md`** - توثيق كامل بالعربي والإنجليزي
2. **`openapi.json`** - يمكن استيراده في Postman أو Insomnia
3. **`endpoints.json`** - البيانات الخام بصيغة JSON
4. **`api_analysis_report.md`** - تقرير تحليلي مفصل

## 📋 أمثلة عملية

### مثال 1: اكتشاف endpoint جديد

```python
# في Python
from scripts.reverse_engineer_api import APIReverseEngineer

engineer = APIReverseEngineer("https://api.fustog.app/api/v1")

# حلل ملف HAR جديد
endpoints = engineer.analyze_har_file("path/to/new_file.har")

# أنشئ التوثيق
engineer.generate_documentation("new_docs.md")
```

### مثال 2: اختبار endpoint

بعد الحصول على `openapi.json`:
1. افتح Postman
2. Import → File → اختر `data/openapi.json`
3. جميع الـ endpoints ستظهر جاهزة للاختبار

### مثال 3: إنشاء TypeScript Types

من البيانات المكتشفة، يمكنك إنشاء types:

```typescript
// types/api.ts
export interface Category {
  id: number;
  name: string;
  // ... باقي الحقول من التحليل
}
```

## 🔍 ما الذي تم اكتشافه حتى الآن؟

من التحليل الحالي، تم اكتشاف:

1. **`GET /category/Categories`** - جلب التصنيفات
2. **`GET /product/ProductsByCategory`** - جلب المنتجات حسب التصنيف
3. **`GET /product/ItemsPrices`** - جلب الأسعار

## 💡 نصائح مهمة

### 1. التقاط حركة مرور شاملة
   - استخدم التطبيق بشكل كامل
   - جرب جميع الميزات
   - سجل كل العمليات (تسجيل دخول، إضافة للسلة، إلخ)

### 2. حفظ ملفات HAR متعددة
   - كل جلسة قد تكشف endpoints جديدة
   - احفظ ملفات منفصلة لكل عملية

### 3. تحديث التحليل بانتظام
   ```bash
   # كلما اكتشفت endpoints جديدة، شغل التحليل مرة أخرى
   python scripts\quick_reverse_engineer.py
   ```

### 4. استخدام النتائج
   - استخدم `openapi.json` في Postman للاختبار
   - استخدم التوثيق لفهم الـ API
   - أنشئ TypeScript types من البيانات
