# Farq — تطبيق تصاميم جديدة مع الحفاظ على الهوية

انسخ هذا المستند أو أعطِه للوكيل عند كل تصميم جديد. المشروع الفعلي: **`platform/`** (Next.js، React، TypeScript، Tailwind v4 عبر `@theme` في `globals.css`، **ESLint** — وليس Biome).

## غير قابل للتفاوض (الهوية والبيانات)

- **المنتج**: Farq كما هو — عربي/إنجليزي، نفس المسارات، نفس التدفقات، نفس استخدام الـ API ما لم يوافق المستخدم صراحة على تغيير السلوك.
- **لا** تعيد تسمية التطبيق، ولا تحذف تدفقات أساسية، ولا تغيّر منطق UX الأساسي إلا إذا التصميم يفرض ذلك **والموافقة موجودة**.
- **ألوان المطاعم/المتاجر تبقى من البيانات**: شعارات السلاسل، ألوان المقارنة في الرسوم البيانية، وأي ألوان تمثل جهة خارجية حقيقية — تبقى كما هي (hex من البيانات أو الـ mapping الحالي). فقط **كروم Farq** (خلفيات، نصوص، حدود، أزرار أساسية، focus، حالات نجاح/تحذير) يمر عبر الـ tokens.
- **بدون تراجع**: غيّر المظهر و`className` قدر الإمكان؛ البنية فقط عند الحاجة.

## مدخلات التصميم

قد تُعطى: Figma، لقطات، PDF، مواصفات نصية، قائمة مكوّنات. **استخرج الـ tokens أولاً** ثم اربط الشاشات بالمكوّنات الموجودة.

مرجع لقطات مجتمع (محليًا بعد فك الضغط): `platform/docs/design-reference/analytics-dashboard-community/`(المجلد مستبعد من Git؛ احتفظ بملف `.zip` محليًا أو أعد الفك عند الحاجة.)

## المنهج التقني (tokens أولاً)

1. حدّث أو عرّف الثيم في `platform/app/globals.css` (`@theme inline` وما يلزم من متغيرات).
2. استخدم Tailwind مع المتغيرات الدلالية بدل hex عشوائي في الكروم.
3. ترتيب الهجرة: tokens أساسية → layout/typography عام → `components/ui/*` → صفحات → حالات خاصة.
4. استبدل الرمادي والـ hex المبعثر في كروم Farq بـ tokens.

## حالات خاصة

- مكتبات تحتاج لون خام: اشتقّ RGB من نفس متغيرات CSS.
- الأيقونات: فضّل `currentColor` + `text-*`.

## النطاق

عدّل فقط ما يتطلبه التصميم. **لا** تضف README إلا إذا طُلب.

## التحقق (من مجلد `platform/`)

```bash
npm run lint
npm run build
```

## قائمة قبول

- التصميم يظهر في الشاشات/المكوّنات المستهدفة.
- كروم Farq يعتمد tokens؛ ألوان البيانات/المتاجر سليمة.
- قلّل `#RRGGBB` في TSX للكروم؛ أي بقايا يجب تبريرها.
- Lint و build ناجحان.

## عند الغموض

- طابق الـ mock في المسافات والمقاييس والزوايا.
- ثبّت أسماء الـ tokens وغيّر القيم فقط.
- سؤال واحد قصير إن حُصِرت؛ وإلا اتبع أنماط المشروع الحالية.
